# Archivepke Morning Dashboard: AI 기반 브랜드 인텔리전스 대시보드

## 문제
> 10명 규모 패션 브랜드 팀이 매일 아침 흩어진 리뷰·경쟁사·트렌드 데이터를 수동으로 확인하느라 핵심 의사결정이 지연된다

- **현재 상태**: 자사몰·29CM·무신사 3개 채널의 리뷰를 각각 확인하고, 경쟁사 뉴스·검색 트렌드·플랫폼 순위를 수동으로 취합. 팀원 10명이 각자 다른 정보 수준으로 아침 회의에 참석. 리뷰 감성 분석은 엑셀 수작업으로 주 1회 가능.
- **원하는 상태**: 매일 아침 한 페이지에서 AI 브리프 3줄 + 리뷰 감성 분석 + 경쟁사 동향 + 검색 트렌드 + 29CM 랭킹을 확인. 팀 전원이 동일한 데이터 기반으로 의사결정.
- **성공 기준**: (1) 아침 데이터 취합 시간 30분 → 대시보드 접속 1분 (2) 리뷰 감성 분석 주기: 주 1회 수작업 → 매일 자동 (3) 팀 10명 중 8명 이상 주 3회 이상 접속

| 2 | 채널별 리뷰 요약 | Google Sheets (Review Clipper) | 채널별 신규 리뷰 수·제품 수 | ✅ 배포 |

## 변화 기록
- **Day 1**: "Claude Code가 뭔지도 몰랐다" → 핵심 기능 7개 체험, 대화형 개발의 가능성을 체감
- **Day 2**: "MCP로 외부 서비스를 연결할 수 있구나" → Slack·Notion·Google Workspace MCP 연동, context-sync 스킬 제작. 대시보드에도 같은 원리(외부 API 연동)가 적용됨을 이해
- **Day 3**: "모호한 요구사항을 구조화하는 것도 자동화할 수 있구나" → 전략투자 도메인 특화 clarify 스킬 제작. PRD를 통해 대시보드 프로젝트를 문서화
- **가장 크게 달라진 점**: 처음엔 "대시보드를 만들어줘"라는 모호한 요구를 던졌지만, Clarify 사고방식을 익힌 후에는 "누구의, 어떤 불편을, 어떤 수치로 해결하는가"를 먼저 정의하게 됨. 기술 구현보다 문제 정의가 먼저라는 것을 체감. 또한 사전 기반 감성 분석의 한계를 발견하고 Claude Haiku 증분 분석으로 전환하는 등, AI를 도구가 아닌 "판단의 파트너"로 활용하는 관점이 생김.
